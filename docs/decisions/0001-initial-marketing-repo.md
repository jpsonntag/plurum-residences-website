# 0001 — Eigenes Marketing-Repo für Plurum Residences

## Entscheidung

Die öffentliche Marketing-Website wird als eigenes Repository `plurum-residences-marketing` unter `drax/client-work` geführt.

## Kontext

Das bestehende `plurum-residences` Repo enthält die operative Produkt-/Verwaltungs-App. Die Marketing-Website hat andere Anforderungen: öffentlicher Auftritt, SEO, Content, Bilder und einfache Pflege.

## Konsequenzen

- klare Trennung zwischen operativer App und öffentlichem Marketing
- spätere Integration über API/CMS möglich
- eigenständiges Deployment der Website möglich
